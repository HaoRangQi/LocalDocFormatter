# DocFormat QA 测试报告（2026-05-30）

## 1. 理解与分析

### 1.1 功能与流程

1. 扫描本地文件或目录，识别可转换文件。
2. 根据源文件类型限制可选目标格式。
3. 可选启用 AI 修正后再导出目标格式。
4. 任务异步执行，前端轮询任务状态。
5. 输出转换结果与 `conversion-report.json/csv` 报告。
6. Docker 模式通过挂载目录完成文件访问与浏览。

### 1.2 核心模型

| 模块 | 关键对象 |
|---|---|
| 转换任务 | `JobManager`、`Job`、`FileResult` |
| 转换引擎 | `LibreOfficeConverter`、`ConversionSpec` |
| AI 修正 | `OpenAICompatibleClient`、`AICorrectionJobManager` |
| 配置与接口 | `AIConfigStore`、`DocFormatApp`、`RequestHandler` |

### 1.3 关键路径与复杂点

| 路径 | 复杂点 |
|---|---|
| `/api/scan` | 大量文件扫描、隐藏文件与锁文件过滤 |
| `/api/jobs` | 逐文件目标格式校验、AI 修正串行处理 |
| `/api/ai/models/refresh` | 第三方错误透传时的敏感信息脱敏 |
| `/api/browse` | Docker 挂载根目录约束、路径越界防护 |

## 5. 代码审查与静态测试

### 5.1 审查结论

| 项目 | 结论 |
|---|---|
| 安全 | 已覆盖 token 校验、路径穿越、key 脱敏；仍需严格本地绑定 |

## 6. 测试执行模拟

### 6.1 自动化执行流程

```bash
python3 -m unittest discover -s tests
python3 -m compileall docformat tests
node --check docformat/web/app.js
docker compose config
docker build -t docformat:local .
```

### 6.2 手工 smoke（Docker 模拟模式）

1. 启动：`DOCFORMAT_CONTAINER=1 DOCFORMAT_WORKSPACE_ROOTS=<path> DOCFORMAT_PORT=38173 python3 -m docformat`。
2. 打开页面，确认状态显示 `Docker + LibreOffice 已就绪`。
3. 点击 `选择文件夹`，弹出路径浏览器。
4. 选择当前目录并确认，验证 `sources` 回填且扫描计数更新。

## 整体测试报告总结

- 当前版本已具备覆盖核心业务流的自动化测试，新增安全、HTTP、性能、Docker、UI 契约测试。
- 已基于测试结果修复 1 个 API key 泄露风险。
- 未完成项仅剩 Docker 镜像真实构建验证，根因是当前机器 Docker daemon 未运行。

## 高优先级风险清单

1. `GET /api/health` 返回 token，若服务暴露到公网会提升被滥用风险。
2. Docker 镜像构建未在当前环境完成，发布前需补充真实构建验收。
3. 关键文件体积偏大，后续迭代风险随功能增加而上升。

## 建议改进点

1. 将 `/api/health` 中 token 移除，仅在本地 `app-config.js` 注入。
2. 增加 Playwright 回归套件覆盖端到端主流程。
3. 引入 `coverage.py` 并设置覆盖率门槛（如 80%）。
4. 拆分 `server.py` 与 `web/app.js`，降低复杂度。
